# Video Motion Design Guide

이 문서는 이 루트 폴더의 모든 Remotion, HyperFrames 영상 작업에서 공통으로 참고하는 디자인 가이드입니다. 특정 작업 폴더나 특정 도구의 소스 파일이 아닙니다.

## 기본 목적

음성이나 스크립트의 핵심 메시지를 짧은 텍스트와 인포그래픽 모션으로 이해시키는 영상을 만든다.

텍스트는 내용을 전부 받아쓰는 자막이 아니라, 장면의 의미를 고정하는 표지판 역할을 한다. 인포그래픽은 텍스트를 보조하는 장식이 아니라 메시지 구조를 보여주는 본체다.

## 화면 톤

- 깨끗한 업무용 브리핑 느낌
- 밝은 캔버스, 진한 잉크 라인, 절제된 포인트 컬러
- 슬라이드 템플릿처럼 정적인 화면보다 흐름도, 분기, 처리 파이프라인, 체크리스트 같은 정보 구조 중심
- 과한 장식, 그라데이션 히어로, 반복 카드 그리드 지양
- 특정 도구 로고에 의존하지 않고, 범용적인 아이콘과 구조로 설명

## 색상

- Canvas: `#F4F7F5`
- Ink: `#17201D`
- Muted ink: `#60716B`
- Structure teal: `#0E8F88`
- Warning coral: `#E45A3D`
- Insight mustard: `#E9B44C`
- Deep violet accent: `#5157C9`

작업 주제에 맞춰 색상은 조정할 수 있지만, 한 가지 색 계열만 반복하는 팔레트는 피한다.

## 타이포그래피

- Primary: `Pretendard`, `IBM Plex Sans KR`, `Malgun Gothic`, sans-serif
- 전체 음성을 자막처럼 계속 노출하지 않는다.
- 장면마다 핵심 제목, 1줄 설명, 짧은 라벨을 사용한다.
- 큰 제목은 1개, 보조 문구는 1개, 라벨은 장면당 2-4개를 기준으로 한다.
- 텍스트는 인포그래픽을 가리지 않아야 한다.
- 모바일/데스크톱 어느 크기에서도 텍스트가 박스 밖으로 넘치지 않도록 줄바꿈과 크기를 관리한다.

## 텍스트 줄바꿈

- 제목과 핵심 문구는 브라우저 자동 줄바꿈에만 맡기지 않고 의미 단위로 줄을 나눈다.
- 한국어 제목에서 한 글자, 조사, 어미, 짧은 술어 어절만 다음 줄에 남기지 않는다.
- 예: `아이디어는 있는데 영상에서 멈춘다면`은 `아이디어는 있는데` / `영상에서 멈춘다면`처럼 나눈다.
- `멈춘다면`, `시작하려면`, `필요합니다` 같은 술어 어절은 단독으로 쪼개지 않는다. 공간이 부족하면 앞 어절과 함께 다음 줄로 보낸다.
- 제목 영역은 `word-break: keep-all`, `overflow-wrap: normal`을 우선하고, 필요하면 제목 컨테이너 폭, 폰트 크기, 행간을 조정한다.

## 인포그래픽 원칙

- 장면마다 서로 다른 구조를 사용한다.
- 같은 카드 배열이나 같은 차트 패턴을 반복하지 않는다.
- 스크립트의 논리 흐름과 화면 구조가 맞아야 한다.
- 문제 제기는 경고, 분기는 맵/노드, 프로세스는 파이프라인, 비교는 전후 구조, 결론은 체크/수렴 구조처럼 의미에 맞는 형태를 고른다.
- 라벨은 도형의 의미를 바로 알 수 있게 붙인다.

## 장면 설계 순서

1. 스크립트를 6-12개 장면으로 나눈다.
2. 각 장면마다 핵심 메시지 1개를 정한다.
3. 핵심 메시지를 제목 1개, 보조 설명 1개, 라벨 2-4개로 압축한다.
4. 앞뒤 장면과 중복되지 않는 인포그래픽 구조를 고른다.
5. 대표 프레임을 렌더해 텍스트 겹침, 여백, 장면 의미를 확인한다.
6. 최종 렌더 전 오디오 길이와 영상 길이가 맞는지 확인한다.

## 하지 말 것

- 전체 음성 자막을 화면에 계속 노출하지 않는다.
- 텍스트만으로 설명하는 슬라이드형 화면을 만들지 않는다.
- 같은 카드 레이아웃을 반복하지 않는다.
- 인포그래픽과 라벨이 서로 겹쳐 의미를 가리지 않게 한다.
- 어두운 배경, 과한 블러, 장식용 오브, 보라/파랑 그라데이션 중심 화면을 기본 스타일로 쓰지 않는다.
