# Decisions

이 파일은 장기적으로 유지할 프로젝트 운영 결정을 기록합니다.

## 2026-05-17

- 루트에는 `README.md`, `AGENTS.md`, `DESIGN.md`, `memory/`, `tools/`를 GitHub 공유 대상으로 둡니다.
- 날짜형 작업 폴더(`YYYYMMDD_작업제목/`)는 로컬 작업공간으로 두되 기본적으로 GitHub에 올리지 않습니다.
- 루트 `README.md`는 GitHub 저장소 방문자가 clone, 의존성 설치, Codex Desktop 플러그인 추가, Remotion/HyperFrames 실행 흐름을 따라가기 위한 온보딩 문서로 유지합니다.
- `DESIGN.md`는 특정 작업 폴더가 아니라 루트에 두고 모든 영상 프로젝트에서 공통으로 참고합니다.
- 새 작업은 `YYYYMMDD_작업제목` 폴더를 만든 뒤 그 안에서 진행하며, 새 작업이 기존 날짜형 작업 폴더를 참조한다고 가정하지 않습니다.
- `memory/`는 자동 저장 기능이 아니라 에이전트가 직접 읽고 업데이트하는 프로젝트 레벨 반자동 메모리로 운영합니다.
- 스크립트 기반 모션그래픽 공통 레이아웃은 상단 설명 카드, 중앙 인포그래픽, 하단 라벨 트레이처럼 안전 영역을 분리하는 방식을 우선합니다.
- 장면별 라벨을 아이콘, 연결선, 카드, 차트 위에 절대좌표로 얹지 않습니다.
- 작업 폴더 루트 `README.md`는 더 이상 만들거나 참조하지 않습니다. 작업 기록이 필요하면 에이전트 응답과 관련 산출물 경로로 충분히 남깁니다.
- 음성 원본 파일은 HyperFrames와 Remotion의 공통 소스이므로 작업 폴더의 `audio/` 안에서 관리합니다. 사용자가 첨부 파일명을 미리 맞출 필요는 없습니다.

## 2026-05-24

- 영상 제작 요청에는 음성 파일과 스크립트 파일을 함께 첨부하는 것을 기본으로 합니다. 첨부 파일명은 자유롭게 두고, Codex가 작업 폴더의 `audio/`와 `script/` 안에 기준 파일로 정리합니다.
- 첨부자료의 브랜드, 톤, 색상, 스타일 단서가 루트 `DESIGN.md`와 겹치면 사용자 명시 요청을 최우선으로 두고, `DESIGN.md`는 공통 제작 원칙, 첨부자료는 작업별 브랜드 보정값으로 다룹니다. 명확한 충돌은 사용자 확인 또는 작업별 디자인 브리프로 기록합니다.

## 2026-05-25

- 새 영상 제작 요청을 받으면 디자인 기준 확인 단계에서 `https://github.com/VoltAgent/awesome-design-md`의 `DESIGN.md` 종류를 리스트업해 사용자에게 선택지를 제시합니다. 선택된 외부 `DESIGN.md`는 작업별 스타일 레퍼런스로만 사용하고, 사용자가 선택하지 않거나 기본값을 요청하면 루트 `DESIGN.md` 규칙을 그대로 사용합니다.

## 2026-05-18

- 프로젝트 루트 폴더명은 `AI Video Studio`입니다. 문서와 예시 경로에서도 이 이름을 기준으로 사용합니다.
- 영상 생성 작업을 병렬화할 때는 Orchestrator가 전체 판단권과 최종 책임을 갖고, Intake/Sync, Design, QA, Render/Packaging Agent를 기본 구성으로 둡니다. Remotion Agent와 HyperFrames Agent는 사용자 요청 또는 Orchestrator 선택에 따라 조건부로 실행합니다.
- 공통 에이전트 프로필 폴더는 루트에 만들지 않고, 작업별 실행 지시가 필요할 때만 날짜형 작업 폴더 안의 `agent-briefs/`에 역할별 brief를 둡니다.
