# Project Memory

이 폴더는 컨텍스트 컴팩트 이후에도 작업 맥락을 빠르게 복구하기 위한 메모리 공간입니다.

## 현재 운영 결정

- 루트 폴더는 영상 제작 작업의 공통 작업실입니다.
- 새 영상 작업은 루트에 바로 만들지 않고 `YYYYMMDD_작업제목` 형식의 로컬 작업 폴더를 만든 뒤 진행합니다.
- 날짜형 작업 폴더는 실행/제작 시마다 새로 생기는 개별 작업공간이므로 기본적으로 GitHub에 올리지 않습니다.
- 루트 `AGENTS.md`는 에이전트가 따라야 할 작업 규칙입니다.
- 루트 `DESIGN.md`는 모든 Remotion, HyperFrames 영상 작업의 공통 디자인 가이드입니다.
- 루트 `README.md`는 GitHub 저장소 방문자를 위한 온보딩 문서로 유지합니다. 에이전트 운영 규칙의 원문은 `AGENTS.md`를 기준으로 합니다.
- 작업 폴더 루트 `README.md`는 만들거나 참조하지 않습니다. 작업별 상세 기록은 최종 응답과 필요한 산출물 경로로 정리합니다.
- 음성 원본 파일은 HyperFrames와 Remotion 공통 소스이므로 각 작업 폴더 루트의 `voiceover.wav`를 기준으로 관리합니다.
- 큰 영상 작업은 Orchestrator 중심의 서브에이전트 구성으로 나눌 수 있으며, Remotion Agent와 HyperFrames Agent는 사용자 요청 또는 Orchestrator 선택에 따라 조건부로 실행합니다.
- 작업별 서브에이전트 지시서는 해당 작업 폴더의 `agent-briefs/`에 둡니다.

## 반복 작업 규칙

- 스크립트 기반 영상 제작에는 음성 파일이 필수입니다.
- 음성 파일이 없으면 제작 전에 사용자에게 추가 요청합니다.
- 인포그래픽과 텍스트는 함께 의미를 만들되 서로 겹치면 안 됩니다.
- 대표 프레임을 렌더해 텍스트 겹침, 여백, 가독성을 확인합니다.
- 타임스탬프가 오디오 내용과 모션그래픽 장면에 맞는지 검증합니다.
- 최종 렌더 전 비디오와 오디오 트랙이 모두 포함됐는지 확인합니다.

## 기록 방식

- 이 폴더에는 프로젝트 운영 결정, 반복 실수 방지 메모, 장기적으로 유지할 규칙만 기록합니다.
- 대용량 산출물, 원본 음성, 렌더 파일, 임시 프레임은 이 폴더에 넣지 않습니다.
- 비밀키, 계정 정보, 개인 인증 정보는 절대 기록하지 않습니다.

## 파일 구성

- `README.md`: 메모리 폴더의 목적과 핵심 운영 원칙
- `decisions.md`: 장기적으로 유지할 결정
- `mistakes.md`: 반복 실수 방지 메모
- `workflows.md`: 반복 작업 절차
